## 定位与北极星（North Star）

**`ghshhf/Operit` 是 Operit 的「工程化进阶版」——上游负责想象，我们负责让它真的能跑。**

我们选择「最可靠的分支」而非「本地优先 / 工具最多 / 纯贡献者友好」，因为：它直接打上游最痛的
结构性短板（无 CI / 无测试 → v1.9.1 整版回退）；本环境能落地；并随测试覆盖增长形成时间复利护城河。

**北极星指标**：用户升级到我们版本后，因回退 / 工具崩溃导致的「用不了」反馈趋近于零。

## 差异化支柱（4 条）

1. **工程护城河（CI + 测试）** — 内部质量：编译门禁 + 回归测试库 + Release 流水线。
2. **工具可靠性（Tool Trust）** — 用户可见质量：统一工具调度层 + 参数校验 + 可观测日志。
3. **贡献者优先（Contributor-First）** — 供给侧飞轮：双语指南 + Dev Container + 架构文档 + 标签体系。
4. **全球可达 + 产品化运营（Reach & Ops）** — 需求侧：i18n + 月度发版（社区 / 文档站**暂缓**，待上游反哺积累记录后再启）。
